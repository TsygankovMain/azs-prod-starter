# Sprint 10: Верхняя отправка отчёта и стабильность фоновой загрузки

Дата: 31.05.2026  
Коммит: `2e60861`

## Что сделано

1. Экран администратора АЗС:
- добавлена верхняя sticky-панель с прогрессом и кнопкой `Сдать отчёт`;
- отправка отчёта перенесена наверх, нижний блок отправки удалён;
- добавлена кнопка `Перейти к проблемам` (скролл к проблемному слоту).

2. Фоновая загрузка фото (frontend):
- вместо fire-and-forget реализована очередь загрузок;
- режим по умолчанию: параллельность `x2`;
- при retryable ошибках Bitrix (`429/504/OPERATION_TIME_LIMIT/timeout`) включается режим `x1`;
- после 10 успешных загрузок в `x1` автоматически возвращается `x2`.

3. Upload API и CRM sync (backend):
- `POST /api/reports/:id/photo` теперь отвечает после `Disk + DB` (без ожидания CRM sync);
- CRM-синхронизация вынесена в фоновую сериализованную очередь по `reportId`;
- добавлены retry/backoff с jitter (`0.8s`, `1.6s`, `3.2s`) для retryable ошибок;
- в ответ `/photo` добавлено `syncQueued: true`;
- для retryable ошибок добавлен структурный `errorCode: bitrix_retryable`.

## Проверка

- backend-тесты целевого набора: пройдены (`50/50`);
- eslint для изменённой frontend-страницы: без ошибок;
- `nuxi typecheck` локально не выполнен из-за отсутствующей optional native-зависимости (`oxc-parser` binding) в окружении npm.
