# Краткое техническое задание к договору

## 1. Наименование работ

Разработка и внедрение Bitrix24-приложения `Фото-отчёт АЗС` для автоматизации сбора и контроля фото-отчётов по объектам сети АЗС.

## 2. Цель работ

Создать приложение, позволяющее автоматически или вручную формировать задания на фото-отчёт, направлять их ответственным сотрудникам АЗС, принимать фото через интерфейс Bitrix24 и предоставлять проверяющим инструмент контроля выполнения.

## 3. Состав функциональности

В рамках работ реализуется:
- встройка приложения в Bitrix24;
- ролевой доступ: администратор, проверяющий, администратор АЗС;
- настройка справочников и полей для АЗС, типов фото и отчётов;
- автоматическое создание отчётов по расписанию;
- ручной запуск отчёта проверяющим по одной или нескольким АЗС;
- уведомления ответственным сотрудникам через бота Bitrix24;
- пошаговая сдача фото в заданной последовательности;
- предпросмотр и пересъёмка фото до подтверждения;
- загрузка подтверждённых фото в Bitrix24 Диск;
- запись ссылки/ID папки с фото в карточку отчёта;
- финальная отправка отчёта сотрудником;
- рабочий стол проверяющего со статусами, фильтрами и быстрыми переходами к отчёту и папке с фото;
- автоматическая фиксация просроченных отчётов.

## 4. Интеграции

Приложение интегрируется с:
- Bitrix24 REST API;
- смарт-процессами Bitrix24;
- Bitrix24 Диском;
- ботом/уведомлениями Bitrix24;
- внешним сервером размещения приложения.

## 5. Требования к размещению

Приложение размещается на выделенной VM Заказчика или иной согласованной инфраструктуре.

Минимальные требования к VM:
- 2 vCPU;
- 4 GB RAM;
- 20 GB SSD;
- Ubuntu Server 22.04 LTS или 24.04 LTS;
- публичный HTTPS-домен;
- Docker и Docker Compose.

Подготовка портала Bitrix24, выдача доступов, настройка домена, VM, HTTPS, backup и сетевых правил выполняются на стороне Заказчика, если иное не согласовано отдельно.

## 6. Результаты работ

Исполнитель передаёт:
- работающее приложение;
- исходный код в согласованном репозитории;
- инструкцию по настройке портала Bitrix24;
- требования к серверу размещения;
- краткую документацию по сценариям использования;
- рекомендации по проверке и эксплуатации.

## 7. Критерии приёмки

Работы считаются выполненными, если:
- приложение открывается из интерфейса Bitrix24;
- администратор может настроить сущности, поля, стадии, роли и расписание;
- проверяющий может вручную создать отчёт по выбранным АЗС;
- автоматический запуск создаёт отчёты по расписанию;
- ответственный сотрудник получает уведомление от бота;
- сотрудник может сделать и отправить обязательные фото;
- фото сохраняются в Bitrix24 Диск;
- карточка отчёта содержит связь с АЗС и папкой фото;
- проверяющий видит статусы выполненных и просроченных отчётов;
- основные сценарии проходят проверку на тестовых данных Заказчика.

## 8. Ограничения

В состав работ не входят, если не согласовано отдельно:
- разработка мобильного приложения;
- доработка штатного мобильного приложения Bitrix24;
- биометрия, геолокация и антифрод-проверки фото;
- интеграции с внешними системами, кроме Bitrix24;
- администрирование инфраструктуры Заказчика после передачи;
- круглосуточная техническая поддержка.

## 9. Допущения

- Заказчик предоставляет доступы к Bitrix24 и необходимые права приложения.
- Заказчик обеспечивает доступность сервера приложения по HTTPS.
- Фото хранятся в Bitrix24 Диске.
- Бизнес-данные хранятся в смарт-процессах Bitrix24.
- Приёмка выполняется на согласованном наборе тестовых АЗС и пользователей.
