# Описание Для Клиента И Продукта

## Назначение

`Фото-отчёт АЗС` помогает регулярно контролировать состояние АЗС через обязательные фото. Приложение само создаёт запросы по расписанию, отправляет уведомление ответственному сотруднику и собирает фото в Bitrix24.

## Кто Использует

- `Администратор` управляет настройками, ролями, расписанием, маппингом полей и видит все разделы.
- `Проверяющий` запускает отчёты вручную, смотрит статусы, просрочки и ссылки на карточки/папки.
- `Администратор АЗС` получает запрос, делает фото по списку и отправляет отчёт.

## Что Входит В MVP

- Единая встройка Bitrix24 с ролевым доступом.
- Смарт-процессы `АЗС`, `Типы фото`, `Отчёты АЗС`.
- Настраиваемый список обязательных фото через справочник `Типы фото`.
- Автоматическая отправка запросов по времени из настроек.
- Ручной запуск отчётов по одной или нескольким АЗС.
- Уведомления от бота `Порядок на АЗС`.
- Пошаговая съёмка фото в строгом порядке.
- Предпросмотр и пересъёмка до отправки фото.
- Фоновая загрузка каждого подтверждённого фото.
- Явная кнопка отправки отчёта после всех фото.
- Рабочий стол проверяющего с фильтрами по дате, АЗС и статусу.
- Запись ID папки Диска в карточку отчёта.

## Основной Сценарий

1. В настройках задано время отчёта.
2. В нужное время приложение создаёт отчёт по активным АЗС.
3. Бот отправляет ответственному сотруднику сообщение с кнопкой открытия отчёта.
4. Сотрудник делает фото по очереди.
5. После каждого фото сотрудник может посмотреть результат и переснять.
6. Подтверждённое фото загружается в фоне.
7. После всех обязательных фото сотрудник нажимает `Отправить отчёт`.
8. Проверяющий видит отчёт в статусе `Выполнено`.

## Ожидаемый Эффект

- Меньше ручных напоминаний сотрудникам.
- Единый контроль выполнения отчётов.
- Фото хранятся в понятной структуре Bitrix24 Диска.
- Проверяющий быстро видит, какие АЗС сдали отчёт, а какие просрочили.

## Ограничения MVP

- Геолокация и антифрод фото не входят в MVP.
- Запрет выбора галереи реализован интерфейсно через камеру, но зависит от WebView/браузера устройства.
- Настройки приложения сейчас хранятся на backend, а не в `app.option`.
- Для scheduler нужен валидный OAuth-контекст администратора портала.
