## Открытые баги

### BUG-001 · HIGH · открыт · Сообщения бота приходят без кнопок
**Симптом (скриншот продакта, 2026-06-11):** личные сообщения бота «Порядок на АЗС» — «Время сделать фото-отчёт… Откройте приложение…» и «Отчёт просрочен. Пожалуйста, укажите причину.» — чистый текст, без кнопки «Открыть приложение» / «Указать причину». Оператор вынужден искать приложение вручную (по ТЗ путь должен быть в один тап).
**След:** `notificationService` поддерживает `keyboard` (кнопки) в `imbot.v2.Chat.Message.send`; в репо есть `dispatchDeepLink` и deep-link причин (коммит d877baf «deep-link триггер причины… нужен e2e в iframe»). Проверить: (а) передаётся ли keyboard в notifyDispatch/notifyExpired вообще; (б) корректен ли формат kebab/BOT_ID (в env `BITRIX_BOT_ID=0` — id резолвится через botRegistry?); (в) не регресс ли после переезда. Начать с логов отправки и одного ручного вызова с keyboard на dev-портале.
**Критерий готовности:** в обоих типах сообщений кнопка, ведущая в приложение (для просрочки — сразу на страницу причины конкретного отчёта).

### BUG-007 · HIGH · открыт · Технические ошибки показываются пользователям как есть
**Симптом (фото продакта, 2026-06-11, телефон оператора):** красная плашка «Ошибка загрузки отчёта — AZS item 164 has empty required photo set field "ufCrm10_1778062997"» — английский текст с внутренним кодом CRM-поля показан оператору АЗС. Требование продакта: ошибки должны быть понятными и легко считываемыми.
**Два уровня фикса:**
1. Конкретный кейс: у АЗС не заполнено поле «Набор фото» в карточке CRM → оператору: «Для вашей АЗС не настроен список фото. Сообщите администратору.»; админу в этой же ситуации — точная диагностика (какая АЗС, какое поле).
2. Системно: бэк возвращает типизированные коды ошибок (errorCode) для известных случаев вместо английской прозы; фронт — словарь код → русский текст + общий fallback «Не удалось загрузить отчёт. Попробуйте позже или сообщите администратору» с технической деталью под спойлером «Подробности» (паттерн error.vue из спринта 2). Прогрепать бэк на другие места, где сырой message уходит в ответ пользователю.
**Критерий готовности:** ни один экран оператора/проверяющего не показывает английский/кодовый текст как основной; техдетали — только под спойлером.
