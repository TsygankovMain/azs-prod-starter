# ТЗ для специалиста Bitrix24: «Фото-отчёт АЗС»

## 1. Подготовка сущностей

Создать или проверить 3 смарт-процесса:
- `АЗС`
- `Типы фото`
- `Отчёты АЗС`

## 2. Поля смарт-процессов

### СП `АЗС`
- `Администратор АЗС` — пользователь (одиночное).
- `Проверяющие` — пользователь (множественное).
- `Набор обязательных фото` — привязка к элементам СП `Типы фото` (множественное).
- `Активна` — да/нет.

### СП `Типы фото`
- `Код фото` — строка (уникальный код, латиница/цифры/`_`).
- `Название фото` — строка.
- `Сортировка` — число.
- `Активность` — да/нет.

### СП `Отчёты АЗС`
Штатные поля:
- `title`
- `assignedById`
- `begindate`
- `closedate`
- `stageId`

Кастомные поля:
- `АЗС` — привязка к СП `АЗС` (или строка).
- `Тип запуска` — строка (`auto`/`manual`).
- `Папка Диска` — **строка** (обязательно, сюда пишется folder id).
- `Загруженные фото` — файл (множественное).

## 3. Стадии отчёта

В СП `Отчёты АЗС` должны быть стадии:
- `Новая`
- `В работе`
- `Выполнено`
- `Просрочено`

Эти стадии нужно сопоставить в `/settings` приложения.

## 4. Права и скоупы приложения

Скоупы приложения:
- `crm`
- `disk`
- `im`
- `imbot`
- `placement`
- `userfieldconfig`
- `user`
- `user_brief`
- `pull`

После обновления скоупов: **переустановить приложение** на портале.

## 5. Бот «Порядок на АЗС»

Требования:
- Зарегистрировать и подключить бота к приложению.
- Бот должен уметь писать в личные сообщения сотрудникам.
- Сообщение от бота должно содержать ссылку на отчёт с форматом:
  - `/marketplace/view/{APP_CODE}/?params[reportId]={id}&params[path]=/admin/{id}`

## 6. Настройки окружения (backend)

Обязательные переменные:
- `BITRIX_REST_ENDPOINT`
- `BITRIX_REST_AUTH_ID`
- `BITRIX_BOT_MODE=bot|notify`
- `BITRIX_BOT_ID`
- `BITRIX_APP_CODE`
- `APP_PUBLIC_BASE_URL`

Рекомендуемый режим: `BITRIX_BOT_MODE=bot`.

## 7. Приёмка

1. Из `/settings` выбираются СП и поля из выпадающих списков портала.
2. Ручной/авто запуск создаёт отчёт и отправляет сообщение от бота.
3. Ссылка из сообщения открывает нужный отчёт `/admin/{reportId}`.
4. После загрузки фото:
   - файл загружается в Диск;
   - поле `Папка Диска` в СП отчёта заполнено строковым folder id;
   - при полном комплекте фото отчёт переходит в `Выполнено`.
5. Для просроченных отчётов отправляются уведомления проверяющему.
