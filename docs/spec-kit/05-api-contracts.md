# 05. API-Контракты

Все пользовательские запросы, кроме install/auth, используют `Authorization: Bearer <JWT>`.

## Auth

### `POST /api/getToken`

Назначение: обменять Bitrix24 OAuth-данные iframe на JWT приложения.

Вход:
- `AUTH_ID`
- `REFRESH_TOKEN`
- `DOMAIN`
- `member_id`
- `user_id`
- `APP_SID`

Проверки:
- `profile` доступен;
- `app.info` доступен;
- `profile.ID` совпадает с `user_id`.

Ответ: JWT и базовые данные пользователя.

### `POST /api/install`

Назначение: обработка установки/переустановки приложения, bot/placement binding.

## Health And Role

### `GET /api/health`

Возвращает состояние API, текущую роль и capabilities.

### `GET /api/me/role`

Возвращает роль текущего пользователя и доступные действия.

## Settings

### `GET /api/settings`

Роль: авторизованный пользователь.  
Возвращает текущие настройки приложения.

### `PUT /api/settings`

Роль: `Администратор`.  
Сохраняет настройки, включая маппинг СП, поля, стадии, расписание и роли.

Ключевые ошибки:
- `400`: невалидный маппинг или пустой обязательный параметр;
- `403`: пользователь не администратор.

## Reports

### `GET /api/reports`

Роли: `Администратор`, `Проверяющий`.  
Возвращает список отчётов с фильтрами по дате, АЗС и статусу.

### `GET /api/reports/summary`

Роли: `Администратор`, `Проверяющий`.  
Возвращает агрегаты для dashboard.

### `GET /api/reports/azs-options`

Роли: `Администратор`, `Проверяющий`.  
Возвращает список АЗС для селектора ручного запуска и фильтров.

### `GET /api/reports/my-active`

Роли: `Администратор`, `Администратор АЗС`.  
Возвращает активные отчёты текущего пользователя.

### `GET /api/reports/:id`

Роли: `Администратор`, `Администратор АЗС`.  
Возвращает отчёт, список обязательных фото и текущий прогресс.

## Manual Launch

### `POST /api/reports/manual`

Роли: `Администратор`, `Проверяющий`.

Вход:
- одна или несколько АЗС;
- плановая дата/время;
- опциональный дедлайн.

Ответ:
- созданные отчёты;
- дубли;
- ошибки по отдельным АЗС.

Ошибки формы возвращаются как `400` с понятными `details`.

## Photo Upload And Submit

### `POST /api/reports/:id/photo`

Роли: `Администратор`, `Администратор АЗС`.  
Загружает одно подтверждённое фото. Проверяет владельца отчёта, размер файла и обязательный `photoCode`.

Поведение:
- создаёт/использует папку Диска;
- загружает файл;
- обновляет служебный статус фото;
- синхронизирует `Папка Диска` и стадию `in_progress` в CRM.

### `POST /api/reports/:id/submit`

Роли: `Администратор`, `Администратор АЗС`.  
Финально отправляет отчёт только если все обязательные фото загружены.

Поведение:
- переводит отчёт в `done`;
- обновляет CRM;
- уведомляет проверяющих.

## Jobs

### `POST /api/jobs/dispatch`

Запускает проверку расписания вручную. Доступ должен быть защищён ролью или `JOB_SECRET`.

### `POST /api/jobs/timeout`

Запускает проверку просрочек вручную. Доступ должен быть защищён ролью или `JOB_SECRET`.
