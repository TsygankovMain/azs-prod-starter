# 03. Настройка Портала Bitrix24

## Смарт-Процессы

Нужно подготовить три смарт-процесса:

- `АЗС`
- `Типы фото`
- `Отчёты АЗС`

Названия могут отличаться, но в настройках приложения нужно выбрать правильные `entityTypeId` и поля.

## СП `АЗС`

| Поле | Тип | Для чего |
|---|---|---|
| `Администратор АЗС` | Пользователь, одиночное | Кому бот отправляет запрос отчёта. |
| `Проверяющие` | Пользователь, множественное | Кого уведомлять о выполнении/просрочке. |
| `Набор обязательных фото` | Привязка к элементам СП `Типы фото`, множественное | Какие фото нужно сделать на этой АЗС. |
| `Активна` | Да/Нет | Участвует ли АЗС в автоматическом расписании. |

Расписание и часовой пояс в карточке АЗС не нужны. Они задаются в настройках приложения.

## СП `Типы фото`

| Поле | Тип | Для чего |
|---|---|---|
| `Код фото` | Строка | Технический код: латиница, цифры, `_`. Используется в имени файла. |
| `Название фото` | Строка | Человеческое название, показывается сотруднику. |
| `Сортировка` | Число | Порядок съёмки фото. |
| `Активность` | Да/Нет | Можно отключить тип фото без удаления. |

## СП `Отчёты АЗС`

По максимуму используются штатные поля:

| Поле | Тип | Для чего |
|---|---|---|
| `title` | Штатное | Название отчёта. |
| `assignedById` | Штатное | Ответственный администратор АЗС. |
| `begindate` | Штатное | Плановое время отчёта. |
| `closedate` | Штатное | Дедлайн/факт закрытия. |
| `stageId` | Штатное | Стадия отчёта. |

Кастомные поля:

| Поле | Тип | Для чего |
|---|---|---|
| `АЗС` | Привязка к СП `АЗС` | Связь отчёта с АЗС. |
| `Тип запуска` | Строка | `auto` или `manual`. |
| `Папка Диска` | Строка | ID папки Bitrix24 Диска, куда грузятся фото. |
| `Загруженные фото` | Файл, множественное | Файлы отчёта, если поле поддерживает нужный формат записи. |

## Стадии СП `Отчёты АЗС`

Нужны стадии и их маппинг в настройках:

- `Новая` -> `new`
- `В работе` -> `in_progress`
- `Выполнено` -> `done`
- `Просрочено` -> `expired`

## Бот

Бот должен называться `Порядок на АЗС`.

Требования:
- бот зарегистрирован через REST Bot Platform/`imbot`;
- бот может писать личные сообщения администраторам АЗС;
- сообщение содержит кнопку открытия отчёта;
- приложение после переустановки имеет актуальные bot/placement permissions.

## Scopes Приложения

Минимально нужны:

- `crm`
- `disk`
- `im`
- `imbot`
- `placement`
- `userfieldconfig`
- `user`
- `pull`

После изменения scopes приложение нужно переустановить на портале.

## Настройка В Приложении

1. Открыть раздел `Настройки`.
2. Выбрать СП `АЗС`, `Типы фото`, `Отчёты АЗС`.
3. Сопоставить поля через выпадающие списки.
4. Если поля нет, создать его кнопкой создания поля.
5. Для СП `Отчёты АЗС` подтянуть и сопоставить стадии.
6. Заполнить роли доступа.
7. Задать времена автоматических отчётов через селектор времени.
8. Сохранить настройки.

## Приёмка Настройки

- В `АЗС` указаны администратор, проверяющие, обязательные фото и активность.
- В `Типы фото` есть активные элементы с кодом, названием и сортировкой.
- В `Отчёты АЗС` есть поле `Папка Диска` типа строка.
- Ручной запуск создаёт отчёт и отправляет сообщение бота.
- После первой загрузки фото в отчёте заполняется `Папка Диска`.
