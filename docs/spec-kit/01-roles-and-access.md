# 01. Роли И Доступ

## Роли

- `Администратор`: полный доступ к настройкам, проверке, отчётам и служебным действиям.
- `Проверяющий`: доступ к рабочему столу проверки, ручному запуску и просмотру отчётов.
- `Администратор АЗС`: доступ только к своим активным отчётам и сдаче фото.

## Правила По Умолчанию

- Администратор портала Bitrix24 получает роль `Администратор`.
- Любой нераспознанный пользователь получает роль `Администратор АЗС`.
- При конфликте списков применяется приоритет: `Администратор > Проверяющий > Администратор АЗС`.

## Настройка Ролей

Роли задаются в настройках приложения через списки user ID:

- `access.adminUserIds`
- `access.reviewerUserIds`
- `access.azsAdminUserIds`

Редактировать эти списки может только пользователь с ролью `Администратор`.

## Матрица Доступа

| Возможность | Администратор | Проверяющий | Администратор АЗС |
|---|---:|---:|---:|
| Открыть настройки | Да | Нет | Нет |
| Менять роли | Да | Нет | Нет |
| Смотреть dashboard проверки | Да | Да | Нет |
| Запускать отчёт вручную | Да | Да | Нет |
| Смотреть свои активные отчёты | Да | Нет | Да |
| Загружать фото в свой отчёт | Да | Нет | Да |
| Отправлять свой отчёт | Да | Нет | Да |
| Запускать jobs вручную | Да | Да | Нет |

## Backend Enforcement

- `PUT /api/settings`: только `Администратор`.
- Reviewer routes: `Администратор` и `Проверяющий`.
- Admin report routes: `Администратор` и `Администратор АЗС`.
- Загрузка и отправка фото дополнительно проверяют, что текущий пользователь является ответственным администратором отчёта или имеет административный доступ.
